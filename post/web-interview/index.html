充电桩运营平台

技术栈：Pug + LayUI + bower + grunt + EasyMock

项目介绍：根据不同运营商的需求，细化运营机制，向设备发送远程控制指令同时接受上报的实时数据，完成对充电业务、充电设备的控制和管理

成果：充电站管理(主要涉及增删查改页面)、站点收费信息(自定义收费和峰谷收费切换)、设备的绑定。在前期后端接口还未开发的情况下，使用EasyMock模拟API，完成前后端数据的调试

车联网管理平台

技术栈：Vue全家桶 + Element + Webpack + Echarts + fetch

项目介绍：完成新增车辆信息的平台注册，在地图车辆监控界面对车辆的电池、整车控制器、充电状态、里程能耗等数据进行实时的监控。 并支持数据删选，可人性化设置需要查看的数据。平台保留所有车辆信息，并对信息进行处理和挖掘。形成电压极差报告、温度极差报告、车辆故障单等。

成果：首页展示车辆全国的分布情况(高德API)，对每辆车的数据进行监控，采用Emqtt连接的方式来实时采集。 threesixty插件完成车辆360度旋转，点击相应零部件展示相关数据信息。通过echarts折线图、柱状图、圆饼图等对后端保存的车辆数据进行展示。
