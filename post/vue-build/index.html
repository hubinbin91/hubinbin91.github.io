vue-cli搭建项目

一、安装相关依赖

Git和Node.js

二、安装vue-cli

cnpm install -g vue-cli

安装若失败，可以使用npm cache clean清理缓存试试

验证安装成功vue -V

三、生成项目

vue init webpack Vue-project

webpack为模板的名称，查看更多的模板(https://github.com/vuejs-templates) Vue-project为项目的名称

四、设置启动

# install dependencies
npm install

# serve with hot reload at localhost:8080
npm run dev

# build for production with minification
npm run build

# build for production and view the bundle analyzer report
npm run build --report

三、Vuex

安装引入Vuex

npm 安装 Vuex

cnpm install vuex -S

main.js中引入

import Vue from 'Vue'
import Vuex from 'Vuex'

Vue.use(Vuex)

new Vue({
  el: '#app',
  store,
  render: h => h(App) 
})

创建store.js文件

Vuex 应用的状态 state 都应当存放在 store.js 里面，Vue 组件可以从 store.js 里面获取状态，可以把 store 通俗的理解为一个全局变量的仓库。

但是和单纯的全局变量又有一些区别，主要体现在当 store 中的状态发生改变时，相应的 vue 组件也会得到高效更新

src目录下创建Vuex目录，新建store.js文件

import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

const store = new Vuex.Store({
  // 定义状态
  state: {
    author: 'admin'
  }
})

export default store

虽然在 main.js 中已经引入了 Vue 和 Vuex，但是这里还得再引入一次

将状态映射到组件

通过在根实例中注册store选项，该store实例会注入到根组件下的所有子组件中，且子组件能通过this.$store访问到

computed: {
  author () {
    return this.$store.state.author
  }
}

修改状态

methods: {
  setAuthor () {
    this.$store.commit('newAuthor',this.data)
  }
}
