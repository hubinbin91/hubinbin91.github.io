let const var关键字

1. let和const

let和const用于创建块级作用域

2. 不存在变量提升 & 暂时性死区

var => 变量可以在声明之前使用，值为undefined

let const => 变量在声明之前使用会报错ReferenceError

用let const声明的变量在声明之前不能获取

if (true) {
  // TDZ开始
  tmp = 'abc'; // ReferenceError
  console.log(tmp); // ReferenceError

  let tmp; // TDZ结束
  console.log(tmp); // undefined

  tmp = 123;
  console.log(tmp); // 123
}

3. 不允许重复声明

var => 后一个覆盖前一个

let const => 再次声明会报错

4. 全局变量不再属于顶层变量

var 声明的全局变量属于window属性

var d = 1;
console.log(window.d); // 1

let g = 1;
console.log(window.g); // undefined

5. let和const区别

let可以先声明再赋值，const声明时必须赋值且为常量

如果是基本类型，存放的就是就是这个变量的值。但如果是引用类型，存放的是它的地址

	return a + b;
}

return a + b;
}

注意： （1）箭头函数内的this指向定义时的对象，而不是调用时的对象 （2）箭头函数不能调用call() apply() （3）箭头函数不能当作构造函数，new时会报错 （4）函数内不存在arguments

ES6中Promise

1. Promise介绍

Promise是异步编程的一种解决方案，用来替代传统的–回调和时间方案。

所谓Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。通过Promise实例调用then()方法表示异步语句执行后要做的事情。

Promise是一个构造函数，自身有：race、all、reject和resolve方法。原型上有：then和catch方法

1.1 基本语法

var p = new Promise(function (resolve,reject) {
    if(// 异步操作成功) {
        resolve(data)
    }
    reject(err)
});
p.then(function(data) {
    // success
}, function(err) {
    // failure
})

简单来说： 1、Promise构造方法接受带有两个参数resolve和reject的匿名函数作为参数 2、resolve将Promise的状态置为成功，并将异步操作结果data作为参数传给成功回调函数 3、reject将Promise的状态置为失败，并将异步操作错误err作为参数传给失败回调函数 4、then方法绑定两个回调函数，分别用来处理Promise成功状态和失败状态。

1.2 Promise状态

Promise对象代表一个异步操作，有三种状态：

  • Pending（进行中）
  • Resolved（已完成，又称 Fulfilled）
  • Rejected（已失败）

1、只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态

2、一旦状态改变，就不会再变。且状态只能从Pending->Resolved或者Pending->Rejected

2. Promise写法

getUserAdmin().then(function(reult) {
    if ( /*管理员*/ ) {
        return getProjectsWithAdmin();
    } else {
        return getProjectsWithUser();
    }
}).then(function(result) {
    /*获取project id列表*/
    return getModules(result.ids);
}).then(function(result) {
    /*获取project id列表*/
    return getInterfaces(result.ids)
}).then(function(result) {
    // ...
})

3. Promise.all()与同时执行一组异步操作

Promise.all接收一个由多个Promise实例组成的数组，如果数组中存在不是Promise的实例则会调用Promise.resolve方法转换。

var fs = require('fs');

function getString(url) {
    return new Promise(function(resolve,reject) {
        fs.readFile(url,function(err,data) {
            if(err) {
                reject(err)
                return
            }
            resolve(data.toString())
        })
    })
}

var arr = [1,2,3];

var promises = arr.map(function(item) {
    return getString('./text' + item + '.txt')
})

Promise.all(promises).then(function(posts) {
    console.log(posts)
})
  • 当数组中所有Promise实例都进入Fullfiled状态，Promise.all实例才会进入Fullfiled状态
  • 当数组中某个Promise实例进入Rejected状态，Promise.all实例进入Rejected状态

Promise.race方法跟Promise.all方法差不多。唯一的区别在于该方法返回的Promise实例并不会等待所有Proimse都跑完，而是只要有一个Promise实例改变状态，它就跟着改变状态。并使用第一个改变状态实例的返回值作为返回值。

4. resolve()函数

我们会在异常操作成功后调用resolve()函数，函数的作用是将Promise的状态由Pending置为Fulfilled，并将异常操作的结果作为参数传递给Fulfilled状态的回调函数

参数类型为Promise时：

var d=new Date();

// 创建一个promise实例，该实例在2秒后进入fulfilled状态
var promise1 = new Promise(function(resolve, reject) {
    setTimeout(resolve, 2000, 'resolve from promise 1');
});

// 创建一个promise实例，该实例在1秒后进入fulfilled状态
var promise2 = new Promise(function(resolve, reject) {
    setTimeout(resolve, 1000, promise1); // resolve(promise1)
});

promise2.then(
    result => console.log('result:', result,new Date()-d),
    error => console.log('error:', error)
)

promise2中调用了resolve(promise1),promise2的状态由promise1决定

5. then()返回一个promise实例

var d = new Date();

var promise1 = new Promise(function(resolve, reject) {
    setTimeout(resolve, 1000, 'resolve from promise1');
});

var promise2 = promise1.then(function(result) {
    console.log('promise1.then(resolve):', result);
});
// promise2 Promise对象

5.1 then()返回值

  • 无论promise1是resolve或者reject，promise2都会只调用成功的回调
  • promise1的then()和promise2的then()同步执行

5.2 then()返回promise

与then()返回值正好相反

5.3 then()无返回

同then()返回值情况相同

总结：在回调方法中一定要有return语句

6. Promise与异常处理

//删除,先检查是否存在，存在再执行真正的删除动作
function del() {
    // 查找
    return find().then(function(resultOfFind) {
        // 如果没找到，直接返回
        if (!resultOfFind) {
            return false;
        }
        // 执行真正做删除操作的方法
        return reallyDelete();
    })
    // ,function(err) {}
}

function deleteItem(req, res) {
    // 删除
    del().then(function(resultOfDelete) {
        // 判断是没找到或者是真正做删除操作的结果，进行处理
        //  ...
        res(resultOfDelete)
    }).catch(function(err) {
        // 处理错误情况
        // ...
        res(err)
    })
}

7. Promise状态透传

如果当前的promise实例没有绑定回调或绑定的不是函数，那么实例就会把其状态以及不可变值或者不可变原因传递给当前实例调用.then方法返回的新promise实例

总结

  1. 回调中一定要使用return；防止丢失状态和结果
  2. 在promise实例的最后使用catch做整体的错误捕获和处理
  3. 利用Promise.then方法对回调函数返回结果的封装
var tic = function(timmer, str){
    return new Promise(function(resolve, reject) {
        setTimeout(function() {
            console.log(str);
            resolve(1);
        }, timmer);
    });
};
 
 
function *gen(){
    yield tic(3000, 'red');
    yield tic(1000, 'green');
    yield tic(2000, 'yellow');
}
 
var iterator = gen();
var step = function(gen, iterator){
    var s = iterator.next();
    if (s.done) {
        step(gen, gen());
    } else {
        s.value.then(function() {
            step(gen, iterator);
        });
    }
}
 
step(gen, iterator);
